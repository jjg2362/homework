# 세 번째 회고
##로그인 과제 회고록
이번 로그인 UI 클론 코딩 과제는 단순한 화면 따라 만들기가 아니라, HTML과 CSS 기본기를 깊이 있게 다지는 과정이었다. 특히 그동안 React와 JavaScript를 활용해서 UI를 다루는 데 익숙했던 나에게는, 순수 HTML과 CSS만으로 UI를 완성해야 한다는 조건 자체가 하나의 도전이었다. 평소 너무나 익숙했던 기술이었지만, 오히려 오랜만에 “기본기만으로 모든 것을 해결해야 하는 상황”이 되니 새롭게 배워야 할 점과 놓쳤던 요소들이 많이 보였다. 이 경험은 친숙한 도구 속에서도 여전히 배울 것이 많다는 사실을 깨닫게 해주었다.

React와 JS로 UI를 구현할 때는 자연스럽게 state, useEffect, 컴포넌트 분리 등의 방식으로 UI 상태를 다뤘다. 하지만 이번에는 순수 HTML과 CSS만으로 기능을 만들어야 했기 때문에 오히려 더 많은 고민을 필요로 했다.

---

이번 과제는 프레임워크 없이 UI를 구현하면서 기본기의 깊이를 다시 다지는 소중한 시간이었다.
겉으로 보기에는 단순한 로그인 화면처럼 보였지만, 실제로는 HTML과 CSS의 근본 원리, 반응형 UI 설계, 접근성, 시멘틱 구조 등 웹 개발의 기초를 다시 정리하는 과정이었다.

이 경험을 통해 **“기본기를 탄탄하게 하는 것이 결국 가장 빠른 성장의 길”**이라는 확신을 얻게 되었고, 앞으로도 이런 과제를 통해 더 깊이 있는 개발자로 성장하고 싶다.
